# JTBD (Jobs to Be Done) 执行技能

基于Alan Klement《When Coffee and Kale Compete》的JTBD理论。本Skill不仅提供理论指导，更具备直接执行能力——可以生成访谈提纲、设计问卷、计算机会分数、输出优先级建议、撰写营销文案等。

## 核心理论速览

JTBD是消费者将现有生活状况转变为更理想状态的过程。客户购买产品不是为了拥有它，而是为了成为"更好的自己"。

**进步力量模型（四种力量）：**
- 推力（Push）：使客户对现状不满的力量
- 拉力（Pull）：吸引客户走向更好生活的力量
- 焦虑（Anxiety）：阻止客户采用新方案的担忧（选择焦虑 + 使用焦虑）
- 惯性（Inertia）：阻止客户改变的习惯力量（选择惯性 + 使用惯性）

**九大原则：**
1. 客户想要进步，不想要产品
2. 人有Job，物品没有
3. 竞争由客户心智定义
4. 采用新方案意味着放弃旧方案（零和）
5. 补偿行为 = 创新机会
6. 重视进步过程而非终点目标
7. 进步定义价值，对比揭示价值
8. 解决方案在使用时刻之外也提供价值
9. 生产者、消费者、解决方案和Job是一个系统

## 执行能力一：生成JTBD访谈提纲

当用户需要做用户访谈、深度访谈、用户研究时，按以下流程执行：

### 输入要求
向用户收集：产品/服务名称、目标用户群体、访谈目的（新客获取/留存优化/流失分析/创新探索）、已知的用户痛点（可选）

### 执行步骤

**第一步：确定访谈对象类型**
根据访谈目的，确定应该访谈哪类用户：
- 新客获取 → 近期开始使用的新客户（30天内）
- 留存优化 → 活跃用户 + 低频用户对比
- 流失分析 → 已取消/停止使用的客户
- 创新探索 → 异常用户 + 补偿行为用户

**第二步：生成访谈提纲**
输出结构化的访谈提纲，包含以下模块（根据目的选择性组合）：

模块A - 挣扎时刻探索（必选）：
- "你第一次是怎么听说[产品]的？"
- "你第一次想到要用这类产品/服务是什么时候？当时生活中发生了什么？"（金矿问题）
- "在使用[产品]之前，你是怎么解决这个问题的？"
- "你什么时候意识到之前的方式不够好了？"

模块B - 竞争格局探索：
- "在尝试[产品]之前，你考虑过哪些其他方案？"
- "你实际用过哪些其他方案？每个方案你喜欢什么、不喜欢什么？"
- "如果[产品]明天消失了，你会怎么办？"

模块C - 推力与拉力分析：
- "是什么让你决定要做出改变？"（推力）
- "你期望使用[产品]后生活会怎么变好？"（拉力）
- "为什么选了[产品]而不是[竞品]？[产品]有什么[竞品]没有的？"

模块D - 焦虑与惯性探索：
- "在第一次使用之前，你有什么顾虑或担心？"
- "是什么阻止你更频繁地使用[产品]？"
- "有没有什么旧习惯让你很难完全切换过来？"

模块E - 进步验证（适用于留存/流失）：
- "使用[产品]后，你的生活/工作有什么具体变化？"
- "你会怎么向朋友描述[产品]？"（自然语言捕捉）
- "有没有什么功能你以为会用但其实没用的？为什么？"

模块F - 功能需求验证：
- "你有没有尝试过自己解决[某个问题]？"（如果没有→该功能可能不值得做）
- "你过去是怎么解决的？用了什么工具或方法？"
- "如果有一个完美的解决方案，它会是什么样的？"

**第三步：输出访谈指南**
为每份提纲附加执行建议：
- 建议访谈时长（通常30-45分钟）
- 关键追问技巧：当受访者说"我觉得..."时追问"你具体做了什么？"（揭示偏好 > 陈述偏好）
- 能量信号识别：标注哪些问题应该特别关注受访者的情绪变化
- 建议样本量和筛选标准

### 输出格式
输出完整的访谈提纲文档，包含：访谈目的、受访者筛选标准、暖场话术、核心问题列表（含追问方向）、结束语、访谈后分析要点。

## 执行能力二：设计JTBD调查问卷

当用户需要做问卷调查、量化研究时，按以下流程执行：

### 输入要求
向用户收集：研究目的、目标人群、已知假设（可选）、预期样本量

### 执行步骤

**第一步：确定问卷类型**
- 筛选型问卷：快速识别哪些人群挣扎最强烈（适合创新探索前期）
- 验证型问卷：验证已发现的JTBD假设（适合定性研究后的量化验证）
- 竞争型问卷：了解用户使用的替代方案和切换原因

**第二步：设计问卷结构**

筛选型问卷模板：
```
Q1. [单选] 你目前在[领域]中面临的最大挑战是什么？
   - 选项A（基于假设的挣扎1）
   - 选项B（基于假设的挣扎2）
   - ...
   - 其他（请填写）

Q2. [量表1-5] 这个挑战对你的影响有多大？
   1=几乎没影响  5=严重影响日常工作/生活

Q3. [单选] 你是否尝试过解决这个问题？
   - 是，投入了大量时间和精力
   - 是，但只是偶尔尝试
   - 没有，因为不知道怎么解决
   - 没有，因为觉得不值得解决

Q4. [多选] 你用过哪些方式来解决？（列出已知的替代方案）

Q5. [开放题] 你注册/购买[产品]时，生活中正在发生什么？
```

验证型问卷模板：
```
Q1. [量表1-5] 以下哪个描述最符合你使用[产品]的原因？
   （将JTBD假设转化为选项）

Q2. [排序] 请按重要性排列以下因素：
   （将进步的各个维度列出）

Q3. [量表1-5] 在使用[产品]之前，以下问题对你的困扰程度：
   （推力强度测量）

Q4. [量表1-5] 在决定使用[产品]时，以下因素的吸引程度：
   （拉力强度测量）

Q5. [量表1-5] 在使用[产品]时，以下因素让你担忧的程度：
   （焦虑强度测量）
```

**第三步：输出完整问卷**
包含：问卷标题、说明文字、所有题目、选项、跳转逻辑、预计填写时长。

## 执行能力三：计算JTBD机会分数

当用户需要评估创新机会、判断哪个Job最值得投入时，执行以下计算。

### 机会分数计算模型

基于JTBD理论设计的机会评估框架，综合四个维度打分：

**维度1：挣扎强度（Struggle Intensity）— 权重30%**
评估客户对现状不满的程度：
- 5分：客户主动投入大量时间精力寻找解决方案，表现出强烈情绪
- 4分：客户有明确的补偿行为（组合多个工具、自制解决方案）
- 3分：客户承认问题存在，偶尔寻找解决方案
- 2分：客户知道问题但接受现状
- 1分：客户未意识到问题或认为不重要

**维度2：替代方案满意度（Alternative Satisfaction）— 权重25%**
评估现有解决方案的满足程度（反向计分）：
- 5分：没有可用的替代方案，或现有方案极差
- 4分：现有方案存在明显权衡，客户被迫妥协
- 3分：现有方案基本可用但有不满
- 2分：现有方案较好，只有小问题
- 1分：现有方案很好，客户满意

**维度3：客户群体规模（Market Size）— 权重25%**
- 5分：大众市场，影响数百万人
- 4分：大型细分市场
- 3分：中型细分市场
- 2分：小众市场
- 1分：极小众

**维度4：预算可获取性（Budget Accessibility）— 权重20%**
客户在这个Job上已经花了多少钱（或愿意花多少）：
- 5分：客户已在替代方案上花费大量金钱（可从多个预算中抢钱）
- 4分：客户有明确预算用于解决此问题
- 3分：客户愿意付费但金额不确定
- 2分：客户习惯免费方案
- 1分：客户不愿意为此付费

### 计算公式
```
机会分数 = (挣扎强度 × 0.30) + (替代方案不满意度 × 0.25) + (市场规模 × 0.25) + (预算可获取性 × 0.20)
```

### 分数解读
- 4.0-5.0：高机会 — 立即投入，这是一个强JTBD
- 3.0-3.9：中等机会 — 值得进一步验证
- 2.0-2.9：低机会 — 需要更多证据或换方向
- 1.0-1.9：不建议 — 挣扎不够强或市场太小

### 执行方式
当用户提供一个或多个待评估的Job时：
1. 逐一询问四个维度的信息（或根据用户提供的研究数据自动评估）
2. 为每个维度打分并说明理由
3. 计算总分
4. 输出排序后的机会优先级矩阵
5. 给出行动建议

## 执行能力四：输出JTBD优先级矩阵

当用户需要对多个Job或多个功能需求排优先级时，执行以下分析。

### 输入要求
用户提供：多个待评估的Job/需求/功能点，以及可用的研究数据

### 执行步骤

**第一步：标准化描述**
将每个需求转化为标准JTBD格式：
```
[动作短语] + [挣扎描述]，这样我就能 + [更好生活的描述]
```

**第二步：四力分析**
对每个Job进行四种力量评估：
| Job | 推力强度 | 拉力强度 | 焦虑程度 | 惯性程度 | 净推动力 |
|-----|---------|---------|---------|---------|--------|
| Job A | ? | ? | ? | ? | (推力+拉力)-(焦虑+惯性) |

净推动力 = (推力 + 拉力) - (焦虑 + 惯性)
净推动力越高，客户采用的可能性越大。

**第三步：机会分数计算**
使用执行能力三的模型为每个Job计算机会分数。

**第四步：输出优先级矩阵**
按以下格式输出：
```
优先级矩阵
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
排名 | Job描述 | 机会分数 | 净推动力 | 建议行动
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 1   | ...    | 4.2     | +3.5    | 立即投入
 2   | ...    | 3.6     | +2.0    | 深入验证
 3   | ...    | 2.8     | +0.5    | 暂缓，收集更多数据
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**第五步：行动建议**
对排名前3的Job给出具体的下一步行动建议。

## 执行能力五：JTBD竞争分析

当用户需要分析竞争格局时，执行以下分析。

### 输入要求
用户提供：产品名称、已知竞品（可选）、用户访谈数据或用户反馈

### 执行步骤

**第一步：识别真正的竞争对手**
基于JTBD原则，竞争不限于同类产品。分析用户在采用本产品前后使用的所有替代方案，包括：
- 直接竞品（功能相似的产品）
- 间接竞品（不同品类但解决同一Job的产品）
- 非消费方案（用户自己的临时解决办法、忍受现状）

**第二步：竞争关系验证**
只有找到实际从一个方案切换到另一个的用户，才能确认竞争关系。输出：
```
确认的竞争关系：
[产品A] <-> [产品B]：N位用户发生了切换
切换方向：A->B (X人) / B->A (Y人)
切换原因：...
```

**第三步：竞争力量分析**
对每个竞争对手进行四力对比：
- 我们的推力优势：客户因为什么对竞品不满而转向我们？
- 我们的拉力优势：我们提供了什么竞品没有的进步？
- 我们的焦虑劣势：客户对我们有什么竞品没有的担忧？
- 我们的惯性劣势：什么习惯阻止客户从竞品切换到我们？

**第四步：输出竞争地图**
用文本格式输出竞争关系图和策略建议。

## 执行能力六：生成JTBD营销文案

当用户需要写产品文案、广告语、落地页文案时，按JTBD原则执行。

### 输入要求
用户提供：产品名称、目标JTBD、文案用途（广告/落地页/社交媒体/邮件）

### 执行原则
1. 用客户自己的语言描述挣扎（不要用行业术语）
2. 帮客户可视化使用产品后的进步（"新我"画面）
3. 直接攻击焦虑（降低采用门槛）
4. 直接攻击惯性（给出切换的理由和路径）
5. 不要描述功能列表，描述生活如何变好

### 文案结构模板

**标题公式：**
- 挣扎共鸣型："还在为[挣扎描述]烦恼？"
- 进步愿景型："[更好生活的描述]，从[产品]开始"
- 对比型："从[旧状态]到[新状态]"

**正文结构：**
1. 开头：描述挣扎时刻（让客户感到"说的就是我"）
2. 中间：展示进步画面（使用产品后的新生活）
3. 消除焦虑：回应常见顾虑
4. 克服惯性：说明切换有多简单
5. 行动号召：明确的下一步

### 执行方式
根据用户指定的用途，生成3个不同角度的文案方案，每个方案从不同的力量切入（推力驱动/拉力驱动/焦虑消除）。

## 执行能力七：JTBD增长与留存策略

当用户需要制定增长策略、降低流失时，执行以下分析。

### 增长策略生成

**第一步：识别增长杠杆**
分析当前用户的进步系统，找到增长机会：
- 上游机会："客户在使用我们之前需要什么？"→ 可以提供的前置服务
- 下游机会："客户实现进步后面临什么新挑战？"→ 可以提供的后续服务
- 横向机会："同一客户还有什么其他Job？"→ 产品组合扩展

**第二步：设计增长飞轮**
```
客户挣扎 → 发现产品 → 实现进步 → 新渴望产生 → 更多产品/服务 → 更大进步
```

### 留存策略生成

**第一步：流失原因分类**
按JTBD理论，流失只有三种原因：
1. Job Done：客户完成了任务，不再需要产品（不一定是坏事）
2. 更好替代：客户找到了更好的解决方案（需要竞争分析）
3. 未实现进步：客户没有感受到进步（产品问题）

**第二步：针对性策略**
- Job Done流失 → 帮客户发现新的Job
- 更好替代流失 → 分析竞品优势，消除权衡
- 未实现进步流失 → 优化新用户引导，帮助建立使用习惯

**第三步：习惯设计**
比较最活跃用户和流失用户的行为差异，找到"啊哈时刻"和关键习惯。输出：
```
最佳用户的关键习惯：
1. [习惯A] — X%的最佳用户在首周内完成
2. [习惯B] — Y%的最佳用户每周执行
建议：引导新用户在前7天内完成[习惯A]
```

## 执行能力八：JTBD描述生成与验证

当用户需要将研究发现整理为标准JTBD描述时，执行以下操作。

### 输入要求
用户提供：访谈记录、用户反馈、或对用户挣扎的描述

### 执行步骤

**第一步：提取关键要素**
从输入材料中提取：
- 挣扎时刻：客户在什么情境下感到不满？
- 推力事件：什么触发了改变的需求？
- 期望进步：客户想象的更好生活是什么样？
- 情感维度：这个Job的情感核心是什么？

**第二步：生成JTBD描述**
使用标准模板：
```
[动作短语] + [挣扎描述]，这样我就能 + [更好生活的描述]
```

生成3个不同表述的版本，从不同角度切入。

**第三步：验证描述质量**
用三个测试问题验证每个描述：
1. 它是否描述了"更好的我"而非某个活动？
2. 它是否也能描述100年前的替代方案？
3. 它是否帮助团队理解应该做什么和不应该做什么？

**第四步：输出"更多关于/更少关于"分析**
```
这个JTBD...
更多关于：[客户真正渴望的情感进步列表]
更少关于：[具体的活动、任务和解决方案列表]
```

## Python 工具包

Skill 提供完整的 Python API，位于 `jtbd/` 包目录下，可直接导入使用。

### 核心模块

| 模块 | 类/函数 | 用途 |
|------|---------|------|
| `analyzer.py` | `JTBDAnalyzer` | 分析引擎：创建JTBD描述、管理四力、生成报告 |
| `interview.py` | `InterviewBuilder` | 访谈框架：按维度生成定制化问题列表 |
| `forces.py` | `ForcesProfile` | 力量分析：推力/拉力/焦虑/惯性结构化分析与诊断 |
| `innovation.py` | `InnovationFinder` | 创新发现：信号识别、机会评估、检查清单 |
| `config.py` | `AnalysisConfig` | 运行时配置：分析维度、输出格式等 |
| `utils.py` | `load_knowledge`, `search_knowledge` | 知识库加载与搜索 |
| `templates.py` | 模板常量 | 访谈问题、报告模板、分析框架模板 |

### 使用示例

```python
import sys
sys.path.insert(0, "/path/to/jtbd-knowledge-skill")
from jtbd import JTBDAnalyzer, InterviewBuilder, ForcesProfile, InnovationFinder

# 分析引擎
analyzer = JTBDAnalyzer("我的项目")
analyzer.add_statement("Help me", "快速找到合适的住处", "专注工作")
analyzer.add_force("push", "频繁出差找酒店耗时", intensity=4)
print(analyzer.generate_report())

# 访谈框架
builder = InterviewBuilder("用户访谈")
builder.include_dimensions(["competition", "push", "anxiety"])
guide = builder.build()
print(InterviewBuilder.render_markdown(guide))

# 四力分析
profile = ForcesProfile()
profile.add("push", "external", "市场竞争加剧", intensity=4)
profile.add("anxiety", "choice", "担心迁移成本", intensity=3)
print(profile.diagnose())

# 创新发现
finder = InnovationFinder()
finder.add_signal("compensating_behavior", "用户用Excel手动追踪")
print(finder.render_markdown())

# 知识库搜索
from jtbd import search_knowledge
results = search_knowledge("焦虑")
```
