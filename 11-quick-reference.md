# JTBD速查手册

## 一、核心概念速查

| 概念 | 定义 | 关键要点 |
|------|------|----------|
| JTBD | 消费者将现有生活状况转变为更理想状态的过程 | 是过程不是需求，描述改变而非活动 |
| 自我提升 | 人类改善自身生活状况的内在渴望 | JTBD的根本驱动力 |
| 新我(New Me) | 客户使用产品后想要成为的更好版本 | 产品是实现"新我"的工具 |
| 进步(Progress) | 客户生活状况的持续改善 | 优先于结果和目标 |
| 创造性毁灭 | 新创新抢走现有产品的客户并取代它们 | 竞争是零和博弈 |
| 进步系统 | 客户、产品、生产者之间的相互依赖系统 | 优化系统整体而非单个部分 |

## 二、进步力量速查

| 力量 | 方向 | 描述 | 示例 |
|------|------|------|------|
| 推力(Push) | 生成需求 | 使客户对现状不满的力量 | 第二个孩子使购物变得困难 |
| 拉力(Pull) | 生成需求 | 吸引客户走向更好生活的力量 | 想象不用到处跑就能买到好食品 |
| 焦虑(Anxiety) | 减少需求 | 对未知的恐惧和担忧 | "这个产品真的有用吗？" |
| 惯性(Inertia) | 减少需求 | 保持现状的倾向和习惯 | 习惯了不计划膳食 |

**焦虑细分：**
- 选择焦虑(Anxiety-in-Choice)：从未使用过产品时的担忧 → 驱走首次客户
- 使用焦虑(Anxiety-in-Use)：使用后对某些质量的担忧 → 驱走回头客

**惯性细分：**
- 选择惯性(Habits-in-Choice)：阻止从一个产品切换到另一个的习惯
- 使用惯性(Habits-in-Use)：使用产品后阻止持续使用的旧习惯

## 三、JTBD描述模板

**基本格式：**
```
[动作短语] + [挣扎描述]，so I can / 这样我就能 + [更好生活的描述]
```

**常用动作短语：** give me / help me / make the / take away / free me / equip me

**示例：**
- "让我从弄清安全产品的压力中解脱出来，这样我就有更多时间享受做父母的乐趣"
- "帮我用来自我尊敬的人的激励性建议走出创新低谷"
- "帮我克服独立创业的孤立和压力，这样我就有动力完成我的产品"
- "装备我管理干预措施所需的工具，这样我就能专注于帮助我的病人"

**检验标准：**
1. 是否帮助团队协同工作？
2. 是否描述了"更好的我"？
3. 是否避免了描述产品或使用方式？
4. 是否也能描述它所替代的解决方案？

## 四、访谈问题速查清单

### 理解竞争
- [ ] 在尝试我们的产品之前，你考虑过哪些其他解决方案？
- [ ] 你实际使用过哪些其他解决方案？
- [ ] 如果我们的产品不可用，你会怎么做？
- [ ] 你对每个解决方案喜欢和不喜欢什么？

### 理解推力
- [ ] 你什么时候意识到现有方案不再有效？
- [ ] 当时你的生活中发生了什么？
- [ ] 你注册/购买时生活中发生了什么？（金矿问题）

### 理解拉力
- [ ] 为什么选择这个方案而不是其他？
- [ ] 你期望生活在拥有正确解决方案后会如何变好？

### 理解焦虑
- [ ] 是什么阻止你更多地使用我们的产品？
- [ ] 在首次使用前你有什么担忧？

### 验证功能
- [ ] 你是否曾尝试解决过这个问题？
- [ ] 你过去是怎么解决的？用了什么工具？

### 理解价值
- [ ] 关于我们的产品，什么最让你印象深刻？
- [ ] 你是怎么向朋友描述我们的产品的？（可以请客户展示发送的消息）

## 五、实操清单

### 启动JTBD研究
- [ ] 确定要研究的产品或领域
- [ ] 识别可能在挣扎的客户群体（先宽后窄）
- [ ] 设计访谈问题（聚焦行为而非意见）
- [ ] 进行6-20个客户访谈
- [ ] 团队一起参与访谈（建立共识）

### 分析JTBD数据
- [ ] 整理"更多关于/更少关于"列表
- [ ] 映射客户尝试过的解决方案时间线
- [ ] 识别四种进步力量（推力、拉力、焦虑、惯性）
- [ ] 比较和对比不同解决方案的共同点和差异
- [ ] 撰写JTBD描述（两部分：挣扎 + 更好的生活）
- [ ] 用替代解决方案测试JTBD描述的有效性

### 应用JTBD到产品
- [ ] 确认产品是否帮助客户实现描述的进步
- [ ] 识别并消除不创造价值的功能
- [ ] 识别并解决选择焦虑和使用焦虑
- [ ] 识别并解决选择惯性和使用惯性
- [ ] 考虑互补产品而非功能堆砌

### 应用JTBD到营销
- [ ] 用客户自己的语言描述挣扎和进步
- [ ] 帮助客户可视化使用产品后的进步
- [ ] 针对焦虑设计减少焦虑的信息
- [ ] 明确产品从哪个预算中抢钱
- [ ] 定位产品相对于客户认为的竞争

### 持续优化
- [ ] 定期与客户交谈，刷新竞争格局
- [ ] 问"接下来是什么？"发现新的挣扎和机会
- [ ] 问"之前是什么？"发现上游机会
- [ ] 比较最佳客户和流失客户的习惯差异
- [ ] 监控补偿行为和非预期使用方式

## 六、常见错误速查

| 错误 | 正确做法 |
|------|----------|
| 将JTBD视为活动或任务 | JTBD描述改变和进步，不是活动 |
| 将Job分为功能型/情感型/社会型 | 每个Job是独特的情感力量组合 |
| 限制竞争在同类产品之间 | 竞争可以来自任何地方 |
| 依赖人口统计学做决策 | 关注客户的状况而非特征 |
| 问客户想要什么 | 问客户做过什么 |
| 只关注需求生成力量 | 同等重视需求减少力量 |
| 通过添加功能来增长 | 通过互补产品和系统来增长 |
| 假设低价就能赢 | 客户最看重的是进步 |
| 相信"非消费者"概念 | 如果有JTBD就一定在用某种方案 |
| 假设产品会永远存在 | 每个产品的增长最终都会停止 |
| 从解决方案开始 | 从理解JTBD开始 |
| 只听客户说什么 | 观察客户做什么 |
