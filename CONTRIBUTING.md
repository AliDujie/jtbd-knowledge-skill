# Contributing to JTBD Knowledge Skill

欢迎贡献！一起完善 JTBD 理论工具包。

## 🌟 为什么贡献这个技能？(Why Contribute?)

- **影响广泛** — JTBD 是产品决策的核心框架，你的贡献将帮助无数产品团队
- **实战导向** — 基于真实业务场景，不是纯理论
- **持续成长** — 与 AliDujie 技能生态系统共同演进
- **社区认可** — 贡献者将在文档中被致谢

## 🤝 如何贡献

### 1. 报告问题

创建 Issue 包含：
- 问题描述
- 复现步骤
- 预期行为
- 相关 JTBD 理论引用

### 2. 提交代码

```bash
git clone https://github.com/your-username/jtbd-knowledge-skill.git
git checkout -b feature/your-feature
# 修改代码...
git commit -m "feat: add new interview question template"
git push origin feature/your-feature
# 创建 Pull Request
```

### 3. 改进文档

- 补充 JTBD 理论说明
- 添加案例分析
- 改进中英文翻译
- 修正理论引用

## 📋 代码规范

- PEP 8 风格
- 完整 docstring
- 类型注解
- 清晰的命名

## 🧪 测试

- 新功能需测试
- 确保分析准确性
- 覆盖率 > 80%

## 💡 贡献方向 (Contribution Areas)

### 高优先级 (High Priority)

- [ ] **新增 JTBD 分析工具** — 如竞争地图、客户旅程时间线
- [ ] **改进四力分析框架** — 添加更多诊断问题和策略建议
- [ ] **添加访谈问题模板** — 针对不同行业/场景的定制化问题
- [ ] **补充创新方法** — 基于 JTBD 的创新技术
- [ ] **优化案例研究** — 添加更多真实业务案例

### 中优先级 (Medium Priority)

- [ ] **工作坊引导指南** — 帮助团队应用 JTBD 框架
- [ ] **焦虑降低策略生成器** — 基于四力分析自动推荐策略
- [ ] **习惯打破战术库** — 帮助用户克服惯性的方法集合
- [ ] **与产品路线图工具集成** — JTBD 到功能的转换矩阵

### 文档改进 (Documentation)

- [ ] 补充 JTBD 理论说明
- [ ] 添加更多案例分析
- [ ] 改进中英文翻译质量
- [ ] 修正理论引用和来源
- [ ] 添加视频教程链接

## 🔗 与生态系统协作 (Ecosystem Integration)

本技能与 AliDujie 技能生态系统中的其他技能紧密协作:

- **[Universal Design Methods](../universal-design-methods/)** — JTBD 访谈可与 UDM 的访谈方法结合
- **[Value Proposition Design](../value-proposition-design/)** — JTBD 洞察直接输入价值主张画布
- **[Web Persona Skill](../web-persona-skill/)** — 基于 JTBD 创建更精准的人物角色

贡献时可考虑如何增强与其他技能的协同效应。
